# /ingest — 源文件入库

将 `raw/` 中的一篇源文件（论文 PDF、文章等）按标准流程摄入 wiki。

## 参数

- `$ARGUMENTS`：要处理的源文件名（必填）。例如 `/ingest attention-is-all-you-need.pdf`

## 执行流程

严格按以下 8 步执行，每步完成后再进入下一步：

### 1. 读取源文件

- 定位 `raw/` 中的目标文件
- PDF：先试 Read 工具；若超过 100MB，用 CLAUDE.md 中定义的 OCR 流程
- 若源文件含图片/图表，单独读取查看

### 2. 讨论关键发现

与用户讨论：
- 这篇论文的核心贡献是什么？
- 哪些发现让你印象深刻？
- 与已有 wiki 知识有什么关联或矛盾？
- 你希望重点强调什么？

### 3. 识别实体与概念

列出源文件中出现的新实体和新概念：
- **实体**：作者、机构、模型名、数据集名等
- **概念**：新方法、新框架、新术语、关键指标等
- 与已有的 `wiki/entities/` 和 `wiki/concepts/` 对比，标记哪些是全新的、哪些需要更新

### 4. 撰写摘要页

在 `wiki/summaries/` 下创建摘要页，必须包含：
- 一句话执行摘要
- 关键发现（3-7 条 bullet）
- 引入的新实体、新概念、新论点
- 与已有 wiki 页面的关联（注明一致、补充还是矛盾）
- 留下的开放问题
- 正确的 YAML frontmatter（type: summary, source: "[[raw/文件名]]"）

### 5. 更新实体页

遍历步骤 3 识别的实体：
- 已存在且信息可补充 → 追加新信息，标注来源
- 不存在 → 创建 stub（status: stub），记录已知信息
- 所有更新的实体页需反向链接到摘要页

### 6. 更新概念页

遍历步骤 3 识别的概念：
- 新洞察 → 补充
- 新概念 → 创建
- 矛盾 → 用 `> [!contradiction]` 标注，记录双方论据

### 7. 更新 index.md

- 为每个新建或重大修改的页面添加/更新索引条目
- 确保链接正确、一句话摘要准确

### 8. 记录日志

在 `log.md` 顶部追加：
```
## [YYYY-MM-DD] ingest | 源文件标题
- 新建页面：[[page1]], [[page2]], ...
- 更新页面：[[page3]], [[page4]], ...
```
