# /ingest — 源文件入库

将论文 PDF 按标准流程摄入 wiki。

## 参数

- `$ARGUMENTS`（可选）：
  - **不传** → 扫描 `raw/artical/` 目录，列出所有 PDF，让用户选择要摄入哪篇
  - **传目录** → 如 `/ingest raw/artical/`，处理该目录下所有未录入的 PDF
  - **传文件** → 如 `/ingest raw/artical/某论文.pdf`，只处理指定文件

## 执行流程

严格按以下 8 步执行，每步完成后再进入下一步：

### 0. 定位与去重

- 根据参数定位目标文件（目录则遍历所有 PDF，单文件则只处理该文件）
- 对每个目标文件，查 `[[wiki/synthesis/文献索引.base]]` 按 DOI / 作者+年份 / 标题去重
- 已存在 → 跳过并告知用户；不存在 → 继续

### 1. 读取源文件

- PDF：先试 Read 工具；若超过 100MB 或 Read 返回空/乱码，用 CLAUDE.md 中定义的 OCR 流程
- 若源文件含图表，单独读取查看

### 2. 讨论关键发现

与用户讨论：
- 这篇论文的核心贡献是什么？
- 哪些发现让你印象深刻？
- 与已有 wiki 知识有什么关联或矛盾？
- 你希望重点强调什么？

### 3. 识别实体与概念

列出源文件中出现的新实体和新概念：
- **实体**：作者、机构、模型名、数据集名等
- **概念**：新方法、新框架、新术语、关键指标等
- 与已有的 `wiki/entities/` 和 `wiki/concepts/` 对比，标记哪些是全新的、哪些需要更新

### 4. 撰写摘要页

在 `wiki/summaries/` 下创建摘要页，必须包含：
- 一句话执行摘要
- 关键发现（3-7 条 bullet）
- 引入的新实体、新概念、新论点
- 与已有 wiki 页面的关联（注明一致、补充还是矛盾）
- 留下的开放问题
- 正确的 YAML frontmatter（type: summary, source: "[[raw/文件名]]"）

### 5. 更新实体页

遍历步骤 3 识别的实体：
- 已存在且信息可补充 → 追加新信息，标注来源
- 不存在 → 创建 stub（status: stub），记录已知信息
- 所有更新的实体页需反向链接到摘要页

### 6. 更新概念页

遍历步骤 3 识别的概念：
- 新洞察 → 补充
- 新概念 → 创建
- 矛盾 → 用 `> [!contradiction]` 标注，记录双方论据

### 7. 更新 index.md

- 为每个新建或重大修改的页面添加/更新索引条目
- 确保链接正确、一句话摘要准确

### 8. 记录日志

在 `log.md` 顶部追加：
```
## [YYYY-MM-DD] ingest | 源文件标题
- 新建页面：[[page1]], [[page2]], ...
- 更新页面：[[page3]], [[page4]], ...
```

### 9. 健康检查触发

每完成 10 篇新录入（不含跳过的去重篇），自动触发 `/lint`：

- 检查 `log.md` 中自上次 lint 以来累积的 `ingest-paper` 条目数
- 累计 ≥ 10 → 执行 `/lint` → 在 log.md 追加 `## [日期] lint | 健康检查（ingest ×10 触发）`
- 累计 < 10 → 告知用户当前进度（如 "已连续录入 7 篇，还差 3 篇触发 lint"）
