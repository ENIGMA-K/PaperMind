# /look-into — 论文精读

对已录入的论文进行深度通读和分析。与 `/ingest`（批量浅层录入）互斥——`/look-into` 不追求速度，追求理解深度。

## 与 /ingest 的分工

| 维度 | `/ingest` | `/look-into` |
|------|-----------|-------------|
| 目标 | 建立索引、元数据补全 | 深度理解、批判分析 |
| 阅读量 | 前 2-3 页 + 摘要 | 全文逐段通读 |
| 产出 | 结构化摘要 (stub) | 精读笔记 (mature) |
| 批量 | 支持（目录遍历） | 一次一篇 |
| reading_status | `ingested` | `read` → `analyzed` |

## 参数

- `$ARGUMENTS`（可选）：
  - **不传** → 列出所有 `reading_status: ingested` 的论文，让用户选择精读哪篇
  - **传标题/作者** → 如 `/look-into Panza 2019`，模糊匹配已有摘要
  - **传文件路径** → 如 `/look-into wiki/summaries/Panza 等 - 2019.md`，直接指定

## 前置条件

- 目标论文必须已录入（`reading_status ≥ ingested`）
- 若论文尚不存在于 wiki → 提示用户先执行 `/ingest`

## 执行流程

### 0. 定位

- 根据参数在 `wiki/summaries/` 中定位目标摘要页
- 若 `reading_status` 已是 `read` 或 `analyzed` → 告知用户已精读，询问是否重新分析

### 1. 全文通读

- **必须**读取完整 PDF（所有页面），不可仅读前几页
- 图表**全部**单独查看，理解每张图/表的内容
- 补充材料（supplementary）也需过目

### 2. 深度拆解

逐项分析并记录到摘要页：

#### 2a. 研究设计审查
- 研究类型、随机化方法、盲法实施
- 纳入/排除标准的合理性
- 样本量计算依据（power analysis）
- 主要终点 vs 次要终点的定义和层级

#### 2b. 方法学评估
- 测量技术的原理、参数设置、重复性
- 统计方法的适用性（如为什么选 Cox 而非 logistic）
- 亚组分析和敏感性分析的预设性（pre-specified vs post-hoc）
- 缺失数据处理策略

#### 2c. 结果解读
- 基线表关键差异及临床意义
- 主要结果：效应量 + 置信区间 + p 值的三角解读（不只看 p）
- 亚组分析的一致性/异质性
- 敏感性分析的稳健性

#### 2d. 讨论与结论评估
- 作者对结果的解释是否过度外推
- 与其他研究的对比（哪些一致、哪些矛盾）
- 临床转化可行性（NNT、风险-获益比、适用人群）
- 作者承认的局限性是否充分，有无遗漏

### 3. 交叉引用

- 遍历 wiki 中**同主题**的已有论文，逐篇对比：
  - 方法和人群是否可比
  - 结论是否一致 → `> [!key-finding]`
  - 结论是否矛盾 → `> [!contradiction]` + 双方论据
- 标记本论文填补了哪个 `> [!gap]`
- 留下本论文引发的新的 `> [!gap]`

### 4. 更新摘要页

在现有内容基础上追加 `## 精读笔记` 段落：

```markdown
## 精读笔记

> 精读日期：YYYY-MM-DD

### 研究设计细节
- ...

### 方法学深度评估
> [!method]
- 统计方法：...
- 测量技术：...

### 关键数据
> [!key-finding]
| 指标 | 数值 | 95% CI | p |
|------|------|--------|---|
| ... | ... | ... | ... |

### 与 wiki 交叉验证
- 与 [[xxx]] 一致：...
- 与 [[yyy]] 矛盾：...  `> [!contradiction]`
- 填补空白：...  `> [!gap]`

### 临床转化
> [!clinical-relevance]
- ...

### 我的批判性思考
- 作者是否过度解读？
- 是否有未考虑的混杂因素？
- 这篇论文的结论在什么条件下可能不成立？

### 写作灵感
> [!thesis-idea]
- ...
```

### 5. 更新 frontmatter

- `reading_status` → `read`
- `updated` → 当前日期
- 若用户明确说「已分析」→ `reading_status` → `analyzed`

### 6. 更新实体与概念

- 补充作者机构信息（若精读中发现了新细节）
- 补充/修正概念页中的技术细节
- 所有更新标注来源为本论文

### 7. 记录日志

在 `log.md` 追加：
```
## [YYYY-MM-DD] look-into | 作者 年份 - 标题
- 精读完成，reading_status: read
- 交叉验证：与 [[page1]] 一致，与 [[page2]] 矛盾
- 新 gap：...
```

## 精读质量标准

一篇合格的 `/look-into` 必须能回答以下问题：
1. 这篇论文的研究设计有哪些优势和硬伤？
2. 主要终点的效应量有多大？临床意义是否大于统计学意义？
3. 这篇论文的结论在什么人群中成立？什么人群中可能不成立？
4. 和 wiki 中已有的同主题论文相比，这篇论文的位置在哪里（增量贡献 / 重复 / 挑战）？
5. 如果我用这篇论文写综述，我会引用它的哪个具体数据？放在哪个论点下？
