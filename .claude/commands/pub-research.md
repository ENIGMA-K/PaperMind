# /pub-research — 学术文献调研

对研究问题进行多数据库并行文献检索，数据源仅限学术数据库。

## 参数

- `$ARGUMENTS`：研究问题（建议包含 PICO 要素和时间范围）
- 示例：
  - `/pub-research 2022-2025 年 PD-1 抑制剂在非小细胞肺癌新辅助治疗中的疗效和安全性`
  - `/pub-research CABG 术后桥血管通畅率的影像学评估方法`

## 数据源

| 数据库 | 方式 | 覆盖 |
|--------|------|------|
| PubMed | E-utilities API | 英文生物医学 |
| Semantic Scholar | API | 全学科，含被引数据 |
| Europe PMC | API | 生物医学，含参考文献链 |
| 百度学术 | WebSearch | 中英文，免费题录 |
| 知网 | WebSearch | 中文期刊/硕博 ⚠️ 无摘要 |
| 万方 | WebSearch | 中文期刊 ⚠️ 无摘要 |
| 维普 | WebSearch | 中文期刊 ⚠️ 无摘要 |

## 执行流程

严格按 CLAUDE.md W11 执行：澄清问题 → 翻译检索式 → 并行搜索 → 去重 → 影响力排序 → **交叉验证** → 生成调研报告 → 统计汇报。

### 交叉验证

对 Top 20 核心文献的关键发现进行跨库比对：
- 多篇独立支持 → 标记共识 `> [!key-finding]`
- 单篇独报 → 标记待验证
- 结论矛盾 → `> [!contradiction]` 标注双方论据
- 中英文文献结论方向不一致 → 显式分析可能原因

### 关键原则

- **先澄清，再搜索**：问题太宽时先收缩范围
- **并行优先**：API 调用同时发出，不串行等待
- **中文数据库诚实标注**：知网/万方/维普只能拿到题录，缺少摘要的文献在报告中标注 `⚠️ 仅题录`
- **复用期刊缓存**：优先查 `[[wiki/synthesis/期刊数据库.base]]`，减少 OpenAlex 调用

### 输出

`wiki/synthesis/文献调研_<主题>_YYYY-MM-DD.md` — 含检索策略、Top 20 表格、主题分布、**交叉验证专节**、完整列表
